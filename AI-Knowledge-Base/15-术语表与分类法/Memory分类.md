# Memory分类

## 1. 目的
Memory 分类用于统一知识库中对不同记忆形态的称呼和边界，避免后续把聊天历史、知识库、任务状态、用户偏好等完全不同的对象都笼统称为 memory。

对 Agent 和 AI 应用来说，Memory 不是单一模块，而是一组作用不同、生命周期不同、治理要求不同的记忆机制。

## 2. 分类原则
Memory 建议优先按以下维度分类：
- 生命周期长短
- 是否与任务绑定
- 是否与用户绑定
- 是否属于结构化状态
- 是否进入长期知识资产

## 3. 短时上下文记忆
### 定义
与当前请求或当前对话强绑定、生命周期较短的上下文信息。

### 典型内容
- 当前对话历史
- 最近几轮交互
- 当前页面或任务上下文
- 临时补充的输入条件

### 特点
- 生命周期短
- 通常直接进入模型上下文
- 更像 session context，而不是长期 memory

## 4. 任务记忆
### 定义
与某个任务执行过程相关的状态、步骤、中间结果和计划信息。

### 典型内容
- 当前任务目标
- 已执行步骤
- 中间工具结果
- 子任务状态
- 重试与失败记录

### 特点
- 通常与 Runtime 和状态机绑定
- 对 replay、resume、durable execution 很重要

## 5. 用户记忆
### 定义
与特定用户或租户长期偏好、历史习惯和常见上下文相关的记忆。

### 典型内容
- 常用页面类型偏好
- 常用组件偏好
- 默认展示方式
- 常见工作空间与应用范围

### 特点
- 长期存在
- 与身份和权限强相关
- 需要治理和可删除能力

## 6. 语义记忆
### 定义
经过结构化沉淀后，可被检索和复用的长期知识资产。

### 典型内容
- 案例总结
- 规则知识
- 常见问题模式
- 复盘结论

### 特点
- 更接近知识库
- 通常不直接作为对话历史使用
- 适合进入 RAG 或长期记忆系统

## 7. 工作记忆
### 定义
用于支撑当前推理和决策的中间表示、草稿、候选集合和临时结构。

### 典型内容
- 当前候选组件列表
- 生成中的 Schema 草稿
- 诊断过程中的原因假设
- planner 的局部计划

### 特点
- 更短期
- 更偏 reasoning / execution support
- 不一定需要持久化

## 8. 外部记忆与内部记忆
从存储位置看，还可以分成：
- 内部记忆：直接放在模型上下文中的信息
- 外部记忆：存储在数据库、缓存、向量库或状态存储中的信息

大多数企业系统真正可靠的 memory，都必须依赖外部记忆，而不能只靠长上下文窗口。

## 9. 为什么要分类
Memory 分类的价值在于：
- 不同类型的 memory 需要不同存储和治理方式
- 不同类型的 memory 进入模型的方式不同
- 权限、脱敏、删除和版本管理要求不同
- 评测和观测指标也不同

## 10. 小结
Memory 分类帮助平台把“记忆”从模糊概念变成可设计对象。只有区分短时上下文、任务状态、用户偏好、语义知识和工作记忆，Agent 和 AI 应用的状态管理才会真正清晰、可控、可扩展。
