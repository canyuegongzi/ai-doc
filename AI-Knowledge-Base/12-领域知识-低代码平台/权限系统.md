# 权限系统

## 1. 定义
权限系统是低代码平台中用于控制“谁可以看到什么、操作什么、在什么范围内操作”的核心治理体系。它不仅影响页面访问和编辑权限，也影响组件配置能力、数据源调用范围、逻辑流执行权限、发布权限、运维操作权限以及 AI 系统中的检索与执行边界。

在 AI 场景下，权限系统不能只理解为传统菜单和接口鉴权。它还必须进入：
- 知识检索过滤
- Tool 可见性裁剪
- Agent 执行授权
- 输出脱敏与审计
- 高风险动作审批

## 2. 它解决什么问题
权限系统主要解决：
- 控制不同身份对平台对象的访问和操作范围
- 防止页面、数据、组件和配置的越权访问
- 为发布、运维、审批等高风险动作建立控制边界
- 为 AI 检索、生成和执行提供明确授权上下文

## 3. 它不解决什么问题
- 不等于租户模型，租户定义边界，权限决定边界内能做什么
- 不等于审批流程，审批是高风险动作的附加控制
- 不仅是 RBAC，也常常需要资源级、字段级和动作级控制

## 4. 权限系统的对象范围
低代码平台里的权限对象通常包括：
- 页面与应用
- 组件物料与模板
- Setter 配置项
- 数据源与数据模型
- 逻辑 DAG
- 发布与回滚动作
- 运维查询与诊断能力
- AI 助手的知识源、工具与 Action

因此，权限系统不能只围绕“页面是否可访问”展开，而要覆盖平台级对象图谱。

## 5. 常见权限维度
一个成熟的平台权限体系通常至少包括：
- 身份维度：用户、角色、组织、系统身份
- 资源维度：页面、组件、数据源、逻辑图、日志等
- 动作维度：读、写、编辑、发布、删除、执行、审批
- 范围维度：租户、工作区、应用、环境、字段、记录
- 条件维度：时间、环境、审批状态、数据标签、风险等级

## 6. 权限模型的常见组合
在企业平台中，通常会组合多种模型：
- RBAC：角色控制常规权限
- ABAC：基于资源属性、环境属性做动态控制
- Resource-level ACL：面向具体资源实例授权
- Data-level filter：面向记录级、字段级过滤

AI 系统中的权限通常不能只靠 RBAC，因为很多场景需要：
- 同一工具对不同资源的执行权限不同
- 同一知识库内容对不同用户显示范围不同
- 同一动作在测试环境和生产环境的授权策略不同

## 7. 与低代码核心对象的关系
权限系统要和低代码核心对象显式关联：
- 页面：可查看、可编辑、可发布、可删除
- 组件物料：可使用、可扩展、可管理
- Setter：可配置哪些字段，哪些字段只读
- 数据源：可查询、可调试、可修改连接配置
- 逻辑 DAG：可编辑、可执行、可审批
- 发布流程：可提交、可审批、可回滚

## 8. 与 AI 的关系
AI 系统会把权限问题放大，因为它具备跨对象检索和跨工具调用能力。因此权限系统必须覆盖以下环节：
- RAG：检索时按身份过滤知识
- Prompt 装配：只注入有权限的上下文
- Tool Discovery：只暴露允许调用的工具
- Tool Execution：执行前做权限校验
- Output：对无权看到的数据做脱敏或拒答
- Audit：记录授权依据和拒绝原因

## 9. Tool 和 Agent 中的权限表达
在 Tool / Agent 场景中，权限应是正式输入，而不是隐式上下文。至少需要表达：
- requesterId
- tenantId / appId / envId
- grantedRoles / permissions
- accessibleResourceScopes
- approvalState

这样 Agent Runtime 才能在执行时做：
- tool visible filtering
- execution authorization
- high-risk escalation
- reasoned denial

## 10. 页面级与字段级权限
低代码平台常见的一个难点是：
- 页面整体用户有权限访问
- 但某些区域、组件、字段、数据列没有权限查看或编辑

因此，权限系统需要支持更细粒度模型，例如：
- 字段只读 / 可写 / 隐藏
- 数据列脱敏展示
- 某些按钮仅在特定角色可见
- 某些逻辑节点仅可由特定身份触发

## 11. AI 场景中的高价值能力
如果权限系统建得足够清晰，AI 可以帮助：
- 解释为什么用户无法访问某页面或按钮
- 解释为什么某次发布被拒绝
- 解释某个工具为何不可见或不可执行
- 在生成页面和逻辑图时自动规避越权配置
- 在问答时自动裁剪无权内容

## 12. 建议沉淀的知识条目
- 权限模型说明
- 角色定义与职责边界
- 资源类型与动作矩阵
- 字段级和数据级权限规则
- Tool 授权策略
- 高风险动作授权策略
- 拒绝原因分类
- 典型越权问题案例

## 13. 常见误区
- 把权限系统理解成只控制菜单访问
- 只做页面级权限，不做字段级和动作级权限
- AI 检索和 Tool 执行没有复用统一权限模型
- Prompt 上下文装配绕过权限过滤
- 日志和回放展示时没有继续执行权限裁剪

## 14. 小结
权限系统是低代码平台和 AI 平台的共同治理底座。它不仅决定用户能否访问页面，也决定 AI 能否检索知识、调用工具、生成结果和执行动作。只有把权限作为平台级正式对象，并贯穿检索、生成、执行和审计全链路，AI 才能真正安全落地。
