# 平台运维问答助手

## 1. 场景定义
平台运维问答助手是指围绕低代码平台的发布、构建、运行状态、日志、告警、配置变更、依赖关系和故障记录，提供可检索、可解释、可追溯运维知识服务的 AI 场景。

它不是泛化聊天机器人，也不是一上来就自动修系统的自治 Agent，而是一类以“查询、解释、汇总、辅助排查”为主的运维增强助手。它重点解决的是：
- 帮用户快速找到运维信息
- 帮用户理解问题发生了什么
- 帮用户缩小排查范围
- 帮用户关联发布、日志、依赖和租户上下文

## 2. 为什么这个场景适合先做
在企业平台里，运维问题高频且重复，很多时候并不是“完全不会修”，而是：
- 信息分散
- 日志难读
- 关联关系不清
- 需要专家经验才能快速判断

平台运维问答助手的价值就在于把分散在发布记录、日志系统、监控平台、告警平台和平台对象模型中的信息组织起来，用更低门槛的方式给出答案和排查线索。

## 3. 典型用户问题
常见问题包括：
- 某个页面最近一次发布为什么失败
- 这个租户今天有没有出现页面白屏
- 某组件版本升级后影响了哪些页面
- 这个页面最近的报错集中在哪个节点
- 为什么某个数据源在生产环境不可用
- 最近 24 小时有哪些高风险告警
- 这个页面和上一个版本相比改了什么

这些问题往往不是单一日志查询，而是多来源信息的组合分析。

## 4. 场景目标
平台运维问答助手的目标通常包括：
- 统一查询入口
- 生成可读的运维解释
- 汇总多源证据
- 给出排查顺序和重点对象
- 在回答中保留证据引用
- 为后续诊断 Agent 和执行型 Agent 提供上游能力

## 5. 适合接入的数据源
这个场景通常需要接入：
- 发布记录
- 构建日志
- 页面运行日志
- trace 链路
- 告警事件
- 指标聚合数据
- 数据源失败记录
- 逻辑 DAG 执行记录
- 组件依赖和版本变更记录
- 历史事故案例库

与普通文档问答不同，这里的知识源大量是动态运行数据，而不是静态文档。

## 6. 推荐的总体架构
一个成熟的平台运维问答助手通常包括：
- Intent recognition：识别是查询、解释、排查还是汇总
- Entity extraction：提取页面、租户、组件、环境、时间范围、错误关键词
- Retrieval：组合日志、告警、发布记录、案例和平台对象检索
- Correlation：按页面、版本、租户、组件、环境串联证据
- Answer generation：输出结论、依据、影响范围和下一步建议
- Escalation：当问题超出问答范围时，引导进入诊断或运维流程

## 7. 与传统运维平台的区别
传统运维平台通常更偏：
- 原始日志与指标展示
- 告警和图表
- 手工过滤与钻取

而平台运维问答助手更偏：
- 面向意图的查询入口
- 自动聚合多源信息
- 输出解释而不是原始数据堆叠
- 把平台对象语义和运维事件绑定起来

## 8. 为什么要和低代码领域模型结合
如果这个场景只接日志，不接平台领域对象，价值会非常有限。因为很多问题都与以下对象有关：
- 页面 Schema
- 组件物料版本
- 数据源连接配置
- 逻辑 DAG
- 发布任务
- 租户与权限模型

例如“页面渲染失败”如果不能关联到 schema 变更、组件版本和最近发布记录，回答通常只会停留在表面现象。

## 9. 回答结构建议
运维问答类回答尤其适合结构化输出，建议至少包括：
- 当前结论
- 关键证据
- 影响范围
- 时间线
- 可能原因
- 建议下一步
- 引用来源

这样可以减少用户在复杂运维信息中反复追问。

## 10. 与诊断 Agent 的关系
平台运维问答助手和更复杂的诊断 Agent 可以分层协同：
- 问答助手：偏查询、汇总、解释
- 诊断 Agent：偏主动收集上下文、生成排查路径、修复建议
- 执行 Agent：偏在边界内执行低风险操作

企业里通常更适合先把问答助手做好，再逐步增加诊断与执行能力。

## 11. 成功指标
建议关注：
- 常见运维问题首轮回答命中率
- 用户平均排查时间是否下降
- 问题升级到人工专家的比例是否下降
- 回答引用覆盖率
- 用户采纳下一步建议的比例
- 重复问题的搜索与问答成功率

## 12. 常见误区
- 把它做成纯日志问答，不接平台对象知识
- 不支持时间范围、环境和租户过滤
- 回答里没有证据和时间线
- 一上来就做自动修复，而不先把查询和解释做好
- 不把高频运维问题沉淀为案例知识

## 13. 小结
平台运维问答助手是企业低代码平台非常实际的一类 AI 场景。它不需要一开始就拥有复杂自治能力，但只要能把发布、日志、告警、依赖和领域对象关系组织清楚，就能显著降低运维信息获取和问题定位的成本，并为后续更复杂的诊断与执行型 Agent 打下基础。
